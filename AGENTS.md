# AGENTS.md

本仓库是本地 ZYTXT 打印客户端的原生 C++/Qt 迁移目标。
所有代理、自动化工具和贡献者都必须遵守本文档中的强制约束。

## 强制约束

- 应用必须保持原生 C++/Qt 架构。禁止在本仓库引入 .NET、C#、Electron 或浏览器运行时依赖。
- 除非有明确迁移说明，否则必须保持 `dotnet-print` 现有本地 HTTP 接口契约兼容。
- 业务逻辑必须远离 UI、托盘和 HTTP 处理器代码。
- 必须使用分层目录：
  - `src/core` 和 `include/sleekpr/core`：纯领域逻辑。
  - `src/infrastructure` 和 `include/sleekpr/infrastructure`：文件系统、打印和操作系统适配。
  - `src/http` 和 `include/sleekpr/http`：本地 HTTP 传输层。
  - `src/app`：Qt 应用启动、托盘生命周期和用户可见外壳。
  - `tests`：自动化测试。
- 新增或修改生产行为前必须先写测试。实现逻辑变化之前，必须先添加或更新对应测试。
- 文件职责必须聚焦。一个文件开始混合无关职责时，必须拆分类或拆文件。
- 所有关键代码必须带中文注释，尤其是业务规则、坐标换算、打印行为、HTTP 兼容和系统集成边界。
- 所有已有注释和新增注释必须使用中文。允许在注释中保留必要的接口名、类名、路径名和技术名词。
- 源码必须使用 UTF-8 编码，禁止把生成的构建产物提交到 git。
- 标签坐标在领域层必须始终使用毫米。只有在渲染、预览或真实打印边界才能转换为像素、打印单位或设备单位。
- 真实打印输出属于兼容性接口。任何纸张尺寸、旋转、边距、DPI 或字体渲染变化，发布前都必须通过预览测试和真实设备冒烟验证。
- 禁止硬编码密钥、令牌、客户凭证或机器专属绝对路径。
- 优先使用 Qt 标准能力。新增第三方依赖前必须说明依赖负责人和引入原因。

## 迁移优先级

1. 先建立可靠的原生外壳：托盘生命周期、启动诊断、本地端口占用处理和日志。
2. 保持现有接口兼容：`/health`、`/settings`、`/printers`、`/print/tag` 和预览接口。
3. 优先把标签规划迁移成确定性的领域代码，再优化渲染和打印输出。
4. 打印适配层必须可替换，后续新增设备能力时不得修改标签业务逻辑。
